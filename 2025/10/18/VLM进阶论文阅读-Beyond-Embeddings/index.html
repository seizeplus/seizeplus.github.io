Beyond Embeddings: The Promise of Visual Table in Visual Reasoning

1.Motivation

  • 传统视觉嵌入对外观很强，但在需要常识/学科背景/可供性时乏力，且多做的是“类别级”检索。论文用可生成的 Visual Table 把与当前图像中具体对象相关的 attributes 与 knowledge显式写成结构化文本，并与视觉嵌入联合供下游推理，从而显著补齐知识缺口。

  • 以向量为中心的表示难以审计和定向修改。论文把场景与对象信息转成可读可编辑的 JSON（scene + objects: category/attributes/knowledge），让人和模型都能查看、删除、增改特定字段，实现可解释、可控的推理流程，并减少幻觉与错误传播。

2.不同尺度对齐的描述

这篇论文设计了一种Visual Table（VT）来获取视觉Table，这个Table是对图片的概括以及对里面物品的属性的汇总（scene + objects: category/attributes/knowledge）。首先来说他要训练一个VT生成器，这个模型输入Image经过视觉特征提取器得到视觉表征，指令通过文本Tokenizer和Embeding后得到文本表征，随后视觉特征通过一个MLP层投射到与文本特征同一个维度上，形成多模态序列，再输入到LLM中。模型分为三个阶段进行训练，

1)输入图片和Caption训练图文对齐能力 ，只训练Connector

2)输入图片和问答/指令跟随，训练指令遵循能力 ，训练Connector和LLM

3)使用预先标记好的VT，继续输入图片和指令，将生成的VT于VT Label进行下一个 token交叉熵损失计算。 我们在训练好VT生成器后，通过输入图片、指令、VT能够实现更好的视觉问答效果

PS: 图里“Designed Prompt → Visual Table Annotations”这是离线采标注（造 VT 标签）的流程，只在 Stage-3 当监督目标，不是训练时的第三路输入。
