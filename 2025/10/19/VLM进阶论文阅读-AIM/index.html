AIM: Adaptive Inference of Multi-Modal LLMs via Token Merging and Pruning

1.Motivation
  • 视觉 token 爆炸：多模态 LLM 往往接收由视觉编码器产生的大量视觉 token，导致推理/训练成本高，限制在资源受限场景与长上下文任务中的应用。
  • 长视频信息密集但 token 冗余严重：帧数一增，token 成倍增长，反而挤占模型处理能力；同时大量视觉 token 实际是冗余的，保留全部 token 并非必要，因而需要自适应推理在保证精度的同时动态控制计算量。
2.整体描述

为了减少视觉Token在VLM进行推理的时候对计算资源产生过多的占用，这篇论文提出了一个简单的、无需训练的思路来解决这个问题。具体来说，它分为三个步骤：

  • LLM 之前的 Token Merging: 在视觉编码器之后和进入 LLM 之前，对视觉 token 依据嵌入余弦相似度进行迭代配对合并。将相邻 token 分成 A/B 两组，为每个 A 在 B 中找最相似者并平均其向量，一次合并最多将 token 数减半，可多轮迭代直至达到目标保留率。视频场景下只在同一帧内合并，避免打乱时间顺序。
  • LLM 内部的 Token Pruning：将合并后的视觉 token 与文本 token 拼接送入 LLM；在每一层，用该层的自注意力权重作为图的邻接矩阵，计算每个 token 的PageRank 重要性分数，留下得分高的token，只剪视觉 token、保留文本 token（剪文本会明显伤精度）。具体的计算公式如下：
